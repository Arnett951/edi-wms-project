Check recent files

sql

FROM dbo.EDI940_Raw
ORDER BY RawId DESC;

Check failed files

sql 
SELECT *
FROM dbo.EDI940_Raw
WHERE ProcessStatus LIKE '%FAIL%'
ORDER BY RawId DESC;

Latest failed ISA

sql 
SELECT TOP 1 ISA_ControlNumber
FROM dbo.EDI940_Raw
WHERE ProcessStatus LIKE '%FAIL%'
ORDER BY RawId DESC;

Latest failed file details

sql 
SELECT TOP 1
    ISA_ControlNumber,
    ErrorMessage,
    FileName
FROM dbo.EDI940_Raw
WHERE ProcessStatus LIKE '%FAIL%'
ORDER BY RawId DESC;

Files waiting to process

sql 
SELECT *
FROM dbo.EDI940_Raw
WHERE ProcessStatus = 'RAW_LOADED';

Parse status counts

sql 
SELECT
    ProcessStatus,
    COUNT(*) AS Qty
FROM dbo.EDI940_Raw
GROUP BY ProcessStatus
ORDER BY ProcessStatus;

Current in-memory rate limit

text
20 AI fallback requests per client IP per hour, per application worker process.

Regex PO and ISA lookups bypass this limit.

Common AI fallback log message

text

AI chat routing flow

text
1. Check ISA regex.
2. Check PO regex.
3. Apply the AI-only rate limit.
4. Send unmatched questions to Claude.
5. Claude selects an approved tool.
6. FastAPI executes the controlled SQL helper.
7. Claude summarizes the tool result.
8. On provider failure, return the normal regex help message.

Important security rule

text
The AI model does not receive database credentials and does not write SQL.

It may only choose from the backend-defined tools:
- lookup_po
- lookup_isa
- list_failed_orders

The FastAPI backend remains responsible for parameterized SQL and authorization.
